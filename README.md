<h3 id="overview">:orange_book: Overview </h3>

Over the years, pets have evolved from a strange and wonderful distraction into an integral part of families around the world. Nothing can compare to the joy of returning home to a loyal companion.
</br>
</br>
But what happens if our little furry best friend <strong>disappears</strong>? Statistics shows that the number of lost pets is <strong>increasing every year</strong>.

<h3 id="goal">:dart: Goal of the project</h3>

The goal of this project is to create a web-based platform that is specifically designed for helping owners find and get to know better their lost pets.

<div align="center">
  <h4 id="pets">:dog: Pets Pages :cat:</h4>
  <p align="left">
    On the <strong>All Pets page</strong>, displayed below, publications can be seen for all registered in the system pets, arranged in descending order by date and time of creation.
    </br>
    </br>
    The publications are such that the following information about a pet can be seen:
    </br>
    ✓ Name and status
    </br>
    ✓ User, along with date and time added
    </br>
    ✓ Short description
    </br>
    </br>
    If a user clicks on the pet's name or on the <i>Read more</i> button the user is redirected to the other page - <strong>Pet Details</strong>, with detailed information about the animal, as well as more photos.
    </br>
    </br>
    In this case the user is logged-in and because of that the <strong>ADD A NEW PET</strong> button is available. Otherwise it is not visible.
  </p>
  <hr>
  <img src="https://github.com/RivaIvanova/PetGuide/blob/main/README%20Images/Pets.PNG" width="800px">
  <p align="left">
  </br>
    In the <strong>Pet Details page</strong> more detailed information is displayed for the pet such as <i>type, color, age, size, location, including district, street and additional information</i> about the location, the <i>full description</i> of the animal and information for the <i>user</i> who added the given pet, as well as <i>additional images</i>.
    </br>
    Again the user is logged-in and because of that the <strong>Send to my email</strong> button is available. Otherwise it is not visible. 
    </br>
    </br>
    By using SendGrid an email message with information about the current pet is being sent to the email, with which the user is registered.
    </br>
    </br>
  </p>
  <img src="https://github.com/RivaIvanova/PetGuide/blob/main/README%20Images/Pets%20Details.PNG" width="800px">
</div>

</br>
</br>

<div align="center">
  <h4 id="articles">:memo: Articles Pages :memo:</h4>
   <p align="left">
    Similar to the <strong>All Pets page</strong>, in the <strong>All Posts page</strong>, publications can be seen for all registered in the system articles (posts), arranged in descending order by date and time of creation.
  </br>
  </br>
   The publications are such that the following information about an article can be seen:
  </br>
  ✓ Title
  </br>
  ✓ User and Category
  </br>
  ✓ Short description
  </br>
  </br>
  Again like in the previous description, if a user clicks on the article's title or on the <i>Read more</i> button the user is redirected to the details page of the article, which is shown in the next image.
  </br>
  </br>
  In this case the user is logged-in and because of that the <strong>ADD A NEW ARTICLE</strong> button is available. Otherwise it is not visible.
  </p>
  <hr>
  <img src="https://github.com/RivaIvanova/PetGuide/blob/main/README%20Images/Articles.PNG" width="800px">
  <p align="left">
  </br>
  When visiting the <strong>Details page</strong> of a specific article, the user can see more detailed information about the article like <i>date and time of creation, added tags, information about the user, who has added the article, as well as more general information about the articles in the system (rigth side)</i>
  </br>
  </br>
  </p>
  <img src="https://github.com/RivaIvanova/PetGuide/blob/main/README%20Images/Article%20Details.PNG" width="800px">

<div align="center">
  <h4 id="events">:partying_face:	Events Pages :balloon:</h4>
    <p align="left">
    On the <strong>Events page</strong>, the different publications about the events are sorted in three categories for non-registered users - <i>Today's events, Upcomming events, Past events</i>, and four for registered users - the previously mentioned plus <i>Volunteer events</i>. 
  </br>
  </br>
  The publications are such that the following information about an event can be seen:
  </br>
  ✓ Date and time of the event
  </br>
  ✓ Event name
  </br>
  ✓ Purpose for it's organization
  </br>
  </br>
  If a user clicks on an event - additional information is being displayed automaticaly, showing description of the event and it's location.
  </p>
  <hr>
  <img src="https://github.com/RivaIvanova/PetGuide/blob/main/README%20Images/Events.PNG" width="800px">
  
  <p align="left">
  </br>
The details page of an event contains information about the activities included in the event, more images and the possibility to become a volunteer if the user is logged-in. Otherwise the <strong>Become a volunteer</strong> button is not visible.
  </br>
  </br>
  </p>
  <img src="https://github.com/RivaIvanova/PetGuide/blob/main/README%20Images/Event%20Details.PNG" width="800px">
